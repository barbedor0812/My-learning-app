学习助手应用需求文档（更新版）
一、问题概述
当前版本存在两个关键缺陷，导致用户数据（文件夹结构、计时记录）无法持久保存，严重影响使用体验。

1. 文件夹与笔记分类管理问题
现象：用户新建文件夹（包括子文件夹）后，将 PDF 或笔记移入该文件夹，过一段时间（通常几分钟后，可能是同步后）文件夹会自动消失，其中所有笔记/PDF 文件又回到了“全部”文件夹中。

影响：用户无法按科目/主题组织学习资料，分类功能完全不可用。

2. 今日任务计时记录丢失问题
现象：在“今日任务”页面点击“开始计时”按钮，计时结束后记录的时间（如学习时长）有时会在页面刷新或同步后丢失。

影响：无法准确追踪每日学习时长，学习进度统计失效。

二、期望行为
2.1 文件夹与笔记分类管理
用户创建的文件夹（包括子文件夹）必须永久保存，不会因同步或页面刷新而丢失。

将笔记或 PDF 移动到某个文件夹后，该文件应始终显示在该文件夹内，直到用户再次移动或删除。

所有设备（电脑、手机）登录同一账号后，看到的文件夹结构和文件分类必须完全一致。

2.2 今日任务计时记录
点击“开始计时”按钮后，计时器应准确记录学习时长（秒或分钟）。

计时结束后，该条学习记录必须保存到云端数据库，并在所有设备上同步显示。

即使应用从后台切回、页面刷新、或重新登录，计时记录也不应消失。

三、问题原因分析（供开发者参考）
根据现象推测，可能的原因包括：

数据模型缺陷：

文件夹与笔记的关联关系未正确设计（例如缺少 folder_id 字段或关联表）。

移动文件时只更新了本地 UI 状态，未同步到 Supabase 数据库。

轮询同步时，从云端拉取的数据覆盖了本地刚刚更新的内容（由于时序问题）。

同步机制冲突：

使用轮询同步，但写入数据库的操作未等待完成，导致后续轮询拉取到旧数据并覆盖。

无并发控制，多个同步请求同时执行造成数据覆盖。

计时记录未持久化：

计时数据仅保存在前端内存中，未调用 Supabase API 写入数据库。

或写入后未正确关联用户，导致后续查询不到。

Supabase 表结构不完整：

可能缺少 folders 表，或者 files 表中缺少 folder_id 字段。

计时记录表（如 study_sessions）可能未正确创建或未配置 RLS。

四、修复要求
4.1 数据库结构调整（如需要）
确认是否存在 folders 表，结构至少包含：

id (uuid, primary key)

name (text)

parent_id (uuid, nullable) — 支持子文件夹

user_id (uuid) — 关联 auth.users

created_at (timestamptz)

确认 files 或 notes 表中存在 folder_id 字段，用于关联文件夹。

确认存在 study_sessions 表（记录计时），结构至少包含：

id (uuid, primary key)

user_id (uuid)

duration_seconds (integer)

date (date)

created_at (timestamptz)

4.2 前端逻辑修复
文件夹创建与移动：

创建文件夹时，必须向 Supabase 的 folders 表插入一条记录，并等待响应成功。

移动文件时，更新对应记录的 folder_id 字段，并等待数据库更新成功。

操作成功后，立即调用一次同步（或等待下次轮询），但需确保不会覆盖刚写入的数据。

计时记录保存：

计时结束后，立即将记录插入 study_sessions 表，并捕获错误。

在页面加载时，从数据库加载今日及历史计时记录，并正确显示。

同步机制优化：

采用“乐观更新”策略：本地操作后先更新 UI，后台写入数据库，若写入失败再回滚。

确保轮询同步时，从数据库拉取的数据不会覆盖未同步的本地操作（例如使用时间戳或版本号比较）。

增加同步锁，防止并发请求互相覆盖。

手机后台恢复处理：

使用 visibilitychange 事件（或现有的同样功能的事件），当页面从后台切回前台时，立即执行一次数据同步，确保显示最新数据。

五、测试验证点
修复完成后，请按以下场景测试：

创建一级文件夹 → 创建子文件夹 → 刷新页面 → 文件夹依然存在。

将 PDF 或笔记移入文件夹 → 刷新页面 → 文件在文件夹内，且不在“全部”中。

在一台设备上创建文件夹并移动文件，另一台设备登录同一账号 → 看到相同的文件夹结构和文件。

点击“开始计时”，学习一段时间后停止 → 记录的学习时长显示在今日任务页面，且刷新页面后依然存在。

多台设备同时登录，在一台设备上移动文件，另一台设备在 30 秒内看到变化（轮询间隔内）。

注意:检查更新完成后的页面不要出现点击页面没有反应的问题。

我想要一个CPA学习app，以下是需求： 2.1 模块一：笔记系统（核心功能） 2.1.1 PDF导入与预处理 功能描述： 用户导入PDF文件，APP将其转换为可编辑文本并支持拆分笔记。 交互流程： 首页点击 【导入PDF】 按钮 从手机文件系统选择PDF文件（如“CPA会计教材-第3章.pdf”） 系统解析PDF，提取文字内容 进入 【笔记拆分界面】： 左侧显示PDF原文预览 右侧显示提取出的文本块 用户可手动调整文本块分割，将PDF内容拆分为多个独立笔记（如将10页PDF拆成5-8个笔记） 确认拆分后，每个笔记进入编辑模式，准备挖空 2.1.2 创建笔记（支持两种方式） 方式A：从PDF导入创建 方式B：手动新建笔记 笔记编辑界面： 标题： [文本框] 可从PDF自动提取章节标题 知识点标签： [标签输入框] 支持手动添加或从文本智能推荐 正文（增强型Markdown编辑器）： 显示从PDF提取或手动输入的文本 挖空交互： 选中文字 → 点击工具栏 【挖空】 → 文字被标记为待填空内容（如：会计信息质量要求包括{{可靠性}}、相关性） 智能挖空（进阶功能）： 点击 【智能挖空】，AI自动识别专业术语、定义、年份、数字等，批量建议挖空点，用户可一键确认或逐一调整 2.1.3 复习笔记（填空练习） 功能描述： 复习时，系统隐藏挖空内容，用户填写后核对答案。 交互流程： 用户打开待复习笔记 正文中所有挖空处显示为 【_____】 输入框 用户在每个填空框中输入答案 点击底部 【核对答案】 按钮 系统比对答案： 正确 → 填空框变绿，显示✓ 错误 → 填空框变红，显示正确答案 用户可选择 【重新填空】 或 【完成复习】 以上是核心需求。下面是拓展需求： 2.2 模块二：学习计划（智能排期核心） 2.2.1 初始设置（课程配置） 功能描述： 用户首次使用或添加新课时的配置向导。 输入项： 课程/科目： [下拉菜单/输入框] 例：会计、审计、财管 总学时： [数字输入框 - 单位：小时] 例：会计基础课共120小时 计划完成日期： [日期选择器] 例：2024年8月25日 常规每日可用时间： [滑动条] 例：2小时 月度特殊时段设置： 【新增核心功能】 点击 【设置特殊日期】 按钮 进入配置页面，可按月设置“特殊时段”及该时段内的可用学习时长 例：选择“每个月” → 设置“26日 至 31日” → 学习时长设为“0.5小时” 系统存储此周期性规则，自动应用于所有月份 2.2.2 计划生成算法 核心逻辑： 基于“总学时”和“每日可用时长（含特殊时段）”，动态分配每日学习任务。 计算步骤： 遍历从今日到计划完成日期的每一天 根据“特殊时段”规则，为每一天标记“可用学习时长” 计算总可用学习时长 = 所有日期可用时长之和 每日任务量 = (当天可用时长 / 总可用时长) × 总学时 实际执行：每天学满当天设定的可用时长，直到学完总学时为止 2.2.3 复习计划生成（艾宾浩斯曲线） 触发条件： 用户完成一个笔记的首次学习（即创建笔记并标记为“已学”） 复习节点： 系统自动为该笔记生成复习任务，时间点为：1天后、2天后、4天后、7天后、15天后、30天后 复习任务独立： 复习任务不计入“总学时”消耗，但与每日学习任务并列展示 2.3 模块三：今日任务（每日首页） 2.3.1 任务看板 顶部提示： 若当天为“特殊日期”，显示温馨提示（如“今日为轻度学习日，请根据精力安排复习”） 今日新学： 显示今日学习目标时长（如“今日新学目标：1.5小时”） 列出建议学习的笔记列表 点击笔记 → 进入该笔记的编辑/学习界面，启动计时器，记录实际学习时长 待复习： 列出今天需要复习的笔记（源于艾宾浩斯算法） 点击笔记 → 直接进入该笔记的“复习笔记（填空练习）”界面 复习任务无时长目标，完成即可 2.3.2 快速记录学习状态 功能描述： 记录每天的学习感受 交互： 任务看板下方提供三个表情按钮 “😊 状态很好” “😐 一般般” “😫 有点难” 2.4 模块四：统计与进度（数据可视化） 2.4.1 学习进度概览 进度条1（时间进度）： “已完成学时 / 总学时” 例：32.5 / 120 小时 进度条2（笔记进度）： “已创建笔记数 / 预估总笔记数” 预估总笔记数可由用户手动输入，或基于PDF页数估算（如每2页一个笔记） 连续学习天数： 显示当前连续完成每日任务的次数 2.4.2 学习日历视图 功能： 以月历形式展示每日学习情况 可视化： 每个日期格子显示：计划时长 / 实际时长（如“2h/2.5h”） 颜色标记： 绿色：任务全部完成 橙色：部分完成 灰色：未完成 特殊日期标记： 用户设置的“特殊时段”用浅灰色底色标注 2.4.3 复习分布图 功能： 展示未来7天的复习任务数量，让用户提前做好准备 展示形式： 柱状图或列表，显示每天需要复习的笔记数量
